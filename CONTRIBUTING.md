# Contributing

このリポジトリへの改善提案や修正は歓迎します。

## 基本方針
- 原則や設計意図が伝わる説明を優先してください。
- 実装例は学習者が追いやすい粒度にしてください。
- 大きな構成変更は `Issue` で背景共有してから進めてください。

## 推奨フロー
1. `Issue` を作成する
2. branch を切る
3. `docs/` 配下を更新する
4. `./.github/skills-config/ebook-build/invoke-build.ps1` を実行する
5. `VALIDATION_CHECKLIST.md` を確認する
6. `Pull Request` を作成する

## レビュー観点
- 原則とサンプルの対応が明確か
- リンク切れや構成崩れがないか
- 誤解を招く設計説明がないか
- 実務に転用しやすい内容か
