# ムード適用
カバーは「構造の明確さ」「依存関係の整理」「保守しやすい設計」を一目で伝えるべきです。外側から内側へ向かう層構造、中央に置かれたドメインの核、そして流れを示す矢印によって、クリーンアーキテクチャの考え方を直感的に表現します。冷静で知的なトーンと、実務に役立つ安心感を両立させる。

# フォント適用
モダンで読みやすいサンセリフ書体を使用します。タイトルは太めで高い視認性を確保し、サブタイトルはやや軽めのウェイトで構造と教育性を強調します。

# コンポジション
テキストコンポジション:
- タイトルとサブタイトルは上部ゾーンに配置し、図解領域と明確に分離する。
- テキストは中央寄せまたは左寄せで整え、設計の秩序感を演出する。
- 図解のレイヤー構造とテキストが自然に接続するように配置する。

ビジュアルコンポジション:
- メインビジュアル: 外側から内側に向かう4層の矩形群（Presentation / Application / Domain / Infrastructure）。各層は明確な境界で分かれ、中央に Domain を配置する。
- 中央部: ドメインを中心にした矢印構成で、依存関係が内側に向かうことを示す。
- 背景: ほのかに入るグリッド線または設計図風のガイドラインで、技術文書らしい秩序感を出す。
- アクセント: 角丸矩形や小さなアイコン（API、DB、ビジネスロジック、テスト）を軽く配置し、実装イメージを補助する。
- レイアウト: 上部20%をテキスト、下部80%をレイヤー構造と依存関係の抽象図へ割り当てる。
- デコレーション: 小さな矢印・境界線・ラベルを使い、構造の流れを視覚化する。

レンダリング注記: 矩形にはクリーンなフラットフィルと細いストロークアウトラインを使用。層の境界は明確にし、グラデーションや派手な装飾は避ける。依存関係の矢印はシンプルで読みやすい構成にする。

タイプ注記: レイヤー構造・矢印・境界線を組み合わせた抽象シェイプを採用する。複雑すぎず、教育的・実務的な印象を保つ。

パレット注記: clean-architectural パレットは、クリーンアーキテクチャの論理性・秩序性・実装への信頼感を伝える。青とティールが構造性を、白とグレーが明瞭さを支える。
